7 // In various files of the Goddard subsystem, there are miscellaneous
8 // unused rodata strings. These can be generated via a printf call that
9 // is stubbed out, but not via define printf(...), as IDO 5.3 C
10 // preprocessor did not support va_arg defines. Goddard, however, did
11 // use gd_printf, which is distinct from this type of stubbed call, as
12 // gd_printf actually is called. This could be because gd_printf could
13 // forward to printf (currently unclear whether it might or might not,
14 // gd_printf is awaiting decompilation), which would indicate goddard was
15 // in the middle of moving these calls to the wrapper function. The unmoved
16 // calls result in the unused rodata strings.

18 #define printf(...) \
19  _Pragma ("GCC diagnostic push") \
